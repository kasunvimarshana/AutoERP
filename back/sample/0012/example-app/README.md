# Example Application — Order Bounded Context

This directory contains a **fully working example** of how to use `laravel-ddd-architect` in a real Laravel application, using an `Order` bounded context as the demonstration domain.

---

## What is demonstrated

| Layer | Files | What it shows |
|---|---|---|
| **Domain** | `Order.php`, `OrderId.php`, `OrderStatus.php` | Aggregate with business rules, immutable VO identity, PHP 8.1 enum |
| **Domain** | `OrderWasPlaced.php`, `OrderWasCancelled.php` | Typed domain events with `toArray()` for event sourcing |
| **Domain** | `OrderException.php` | Named exception factory methods for every invariant |
| **Domain** | `OrderRepositoryInterface.php` | Business-driven contract — only query methods that exist in use-cases |
| **Domain** | `OrderIsActive.php` | Composable specification |
| **Application** | `PlaceOrderCommand.php`, `PlaceOrderHandler.php` | Full CQRS write side with DB transaction + event dispatch |
| **Application** | `CancelOrderCommand.php`, `CancelOrderHandler.php` | Domain exception propagation through the command pipeline |
| **Application** | `GetOrderQuery.php`, `GetOrderQueryHandler.php` | CQRS read side |
| **Infrastructure** | `OrderModel.php` | Pure Eloquent model — no business logic |
| **Infrastructure** | `EloquentOrderRepository.php` | Full `toDomain()` / `toPersistence()` mapping |
| **Infrastructure** | Migration | UUID PK, money in minor units, soft deletes |
| **Infrastructure** | `OrderInfrastructureServiceProvider.php` | Bind interface → implementation, load migrations, wire event listeners |
| **Presentation** | `OrderController.php` | Thin HTTP adapter delegating to handlers |

## How this was generated

The entire skeleton was created with **four commands** after installing the package:

```bash
# 1. Scaffold the full Order context
php artisan ddd:make-context Order

# 2. Add the business-specific command/handler pair
php artisan ddd:make-command-handler Order PlaceOrder
php artisan ddd:make-command-handler Order CancelOrder

# 3. Add the query handler
php artisan ddd:make-query-handler Order GetOrder
```

The generated stubs were then filled in with the domain logic you see here.
The package handles all directory creation, namespace injection, and `.gitkeep` files automatically.

## Key architectural decisions illustrated

### 1. Money in minor units (integers, not floats)
```php
// WRONG — floating point precision errors
$total = 10.99 + 5.01; // might not equal 16.00

// RIGHT — store cents as integers
$total = Money::of(1099, 'USD')->add(Money::of(501, 'USD')); // exactly 1600 cents
```

### 2. UUID identity generated by the domain, not the database
```php
// The domain creates the ID — the database just stores it
$order = Order::place($customerId); // OrderId::generate() called internally
$this->repository->save($order);    // UUID already set, DB does nothing special
```

### 3. Domain events dispatched AFTER the transaction commits
```php
DB::transaction(function () use ($command) {
    $order = Order::place($command->customerId);
    $this->orders->save($order);

    // Events pulled INSIDE the transaction, dispatched after commit
    foreach ($order->pullDomainEvents() as $event) {
        Event::dispatch($event); // Laravel queues/processes after commit
    }
});
```

### 4. Reconstitution never fires domain events
```php
// Create path — fires OrderWasPlaced event
$order = Order::place($customerId);

// Reconstitution path — no events (entity already exists)
$order = Order::reconstitute($id, $customerId, $status, $placedAt);
```

### 5. Controllers are completely thin
```php
// BAD — business logic in controller
public function store(Request $request) {
    if ($request->amount < 0) { throw ... } // business rule!
    $order = new Order(...);                  // construction logic!
}

// GOOD — controller is just a pipe
public function store(PlaceOrderRequest $request): JsonResponse {
    $orderId = $this->placeHandler->handle(
        PlaceOrderCommand::fromArray($request->validated())
    );
    return response()->json(['order_id' => $orderId], 201);
}
```

## Running the example tests

Copy the example files into a fresh Laravel 11 application, then:

```bash
# Run just the domain unit tests (no DB needed, instant)
php artisan test tests/Unit/Order/

# Run the full feature test suite
php artisan test tests/Feature/Order/
```
